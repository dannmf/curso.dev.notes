# Construção de um software

Antes de começarmos a abordar a construção organica e a impressora 3D, é importante darmos enfasê que, um software deve ser construído aos poucos, e nunca de uma única vez.

## Modelo Orgânico

No modelo orgânico, o sistema cresce como um organismo vivo: ele é construído por partes funcionais e independentes, como se fossem órgãos ou módulos. Cada componente já nasce com alguma utilidade, pode ser testado isoladamente e, ao ser integrado, melhora o todo.
Esse modelo traz duas vantagens importantes:

Motivação contínua — você vê progresso real rapidamente, porque partes do sistema já começam a funcionar antes do todo estar pronto.
Menos riscos — problemas são identificados cedo, enquanto ainda é fácil ajustar ou corrigir.
É como montar um corpo que vai ganhando coração, pulmões, olhos… cada parte já tem vida própria.

Ele evita o **Over Engineering** que é a construção de um software com muitas funcionalidades desnecessárias ou o **Feature Creep** que é quando um projeto tem uma quantidade tão grande de recursos que acaba atrapalhando.

## Impressora 3D

No modelo impressora 3D, o sistema só aparece como um todo quando a “impressão” termina. Você trabalha muito tempo sem ver nada funcionando, porque tudo depende da conclusão completa da estrutura.
As consequências típicas são:

Baixa motivação — o time só vê o resultado final depois de muito esforço invisível.
Mais riscos acumulados — qualquer erro só é percebido no final, e corrigir pode exigir recomeçar ou refazer grandes partes.
É como imprimir uma peça grande: só no final você descobre se funcionou.
